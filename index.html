UI Constitution

One command to build better, accessible interfaces with AI agents.

Install for Cursor
curl -fsSL https://ui-constitution-a0462cfa89aa.herokuapp.com/install.sh | sh

Usage

/ui

That's it. The agent applies all rules automatically when building or reviewing UI.

Update

Run the same install command again to get the latest version. It detects existing installs and updates in place.

What it enforces

  • → WCAG accessibility requirements
  • → Keyboard and screen reader support
  • → Performance-safe animations only
  • → Accessible component primitives
  • → Tailwind best practices
  • → Opinionated design taste

Changelog

4.0.0

Core systems & opt-in focus rings

Added semantic spacing, elevation, color role, container, component size, and icon size scales. High-visibility focus outlines are now opt-in instead of required by default. Greyscale palette standardized to neutral-only.

3.2.0

Inter font & dark mode

Inter is now the only allowed font. Full dark mode rules with required light/dark color pairing for every surface, text, and border utility.

3.1.0

Border radius constraint

Restricted border radius to subtle values. Banned rounded-xl, rounded-2xl, and rounded-3xl for cards, buttons, and containers.

3.0.0

Explicit Tailwind utilities

Defined allowed values for layout, animation, and design. Banned arbitrary values for z-index, sizing, spacing, shadows, opacity, and colors.

2.2.0

Typography utilities

Explicit allowed font sizes, weights, and line heights. Banned arbitrary typography values.

2.1.0

Typography scale

Added core type scale with defined roles from Display to Code. Every text element must map to a role.

2.0.0

Simplified command & versioning

Consolidated to a single /ui command. Added update detection so re-running the installer upgrades in place.

1.0.0

Initial release

Core accessibility, component, interaction, and performance rules with install script and landing page.
